**Область видимости переменной** и **контекст переменной** — это **участок кода**, в котором **эта переменная доступна (видна)** и может быть использована.

```csharp
        static void Main(string[] args)
        {
	        int a  = 5;
            for (int i = 0; i < 10; i++)
            {
                Console.WriteLine(i);
            }
```
`i` существует только в контексте цикла for.
`a` существует только в контексте метода `Main`

>***Контекст*** есть и у методов.

Например, `myMethod` возвращает `b`, к которой можно обратиться из другого метода.

```csharp
        static int myMethod ()
        {
            int b = 0;
            return b;
        }
```

Но тут `b` существует только в методе, потому что  это `void`, и он не возвращает ничего

```csharp
        static void myMethod ()
        {
            int b = 0;
            
        }
```

>***Контекст*** существует и у классов.

Можно объявить переменную классе `Program`, но чтобы обратиться к такой переменной, нужно создать объект класса.

## Конфликты областей видимости

Мы не можем создать 2 переменные с одинаковым именем в одном контексте видимости.

Интересна вот такая ситуация с одинаковым названием переменной
```csharp
    internal class Program
    {
        static int b = 2;
        
        static void myMethod ()
        {
            int b = 1;
            
        }
	}
```

>Внутри метода локальные переменные **имеют приоритет** над внешними

то есть будет использована `b` внутри метода, если мы попытаемся вызвать ее на консоль в `Main`
