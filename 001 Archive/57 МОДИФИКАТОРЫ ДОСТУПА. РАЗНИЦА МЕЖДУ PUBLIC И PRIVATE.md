### Как работают  модификаторы`private` и `static`

>Модификаторы `private` и `static` нужны для #инкапсуляция


```csharp
class Point
{

    public int x; // public

    private int y = 11; // private

    private void PrintX()
    {
        Console.WriteLine($"X: {x}");
    }
    public void PrintY()
    {
        Console.WriteLine($"Y:{y}");
    }
    public void PrintPoint()
    {
        PrintX();
        PrintY();
    }
}
    internal class Program
    {
        static void Main(string[] args)
        {
        var p = new Point();
        p.x = 1; /// потому что public
        
        }
}
```

> Разница между `private` и `static` в доступности на уровне экземпляра класса

>То есть мы можем вызвать поле `p.x` в `Main` потому что это `public` и не можем вызвать `p.y` потому что у него модификатор `static`.

> Поле `y`существует, просто мы не можем к нему обратиться на уровне объекта.

> При этом мы можем вызвать `p.PrintY();` и вывести значение `private` поля `y`

>С помощью этого реализуется #инкапсуляция 

>Метод `PrintX();` - `private`, хотя поле `x` `public`

### Модификатор доступа по умолчанию

---------
>Если мы не указываем модификатор, то используется `private`
>принято указывать модификаторы в любом случае

### Рефлексия

>Рефлексия нужна чтобы проверить модификаторы полей и методов класса

Добавляем: 
```csharp
using System.Reflection;
```

> Вызываем в `Main`

```csharp
            var typeInfo = typeof(Point).
                GetFields(BindingFlags.Instance |
                BindingFlags.NonPublic |
                BindingFlags.Public);
```


> В результате мы получаем массив, который содержит информацию о полях

> перебираем массив:

```csharp
            foreach (var item in typeInfo)
            {
                Console.WriteLine($"Name {item.Name}
                \tIs public:{item.IsPublic}
                \tIs private: {item.IsPrivate}");
            }
```
