# ИНИЦИАЛИЗАЦИЯ ОБЪЕКТОВ КЛАССА

>ИНИЦИАЛИЗАЦИЯ ОБЪЕКТОВ КЛАССА это процесс создания объектов класса

> Мы уже рассматривали как это происходит:

(`class Cat`)
```csharp
    class Cat
    {
        public int Age { get; set; }
        public string Name { get; set; }

        public Cat()
        {
			
        }
        public Cat(string name)
        {
            Name = name;
        }
    }
```

(типичная инициализация объектов класса)
```csharp
        static void Main(string[] args)
        {
            Cat cat1 = new Cat();
            cat1.Age = 3;
            cat1.Name = "Тимофей";
        }
```

> Мы используем конструктор без параметров и указываем поля возраст и имя уже после инициализации.
# Специальный синтаксис инициализации объектов

>Мы можем заполнить поля класса прямо при его создании.

>Такая инициализация похожа на инициализацию массива.

```csharp
            Cat cat2 = new Cat
            {// фигурные скобки
                Age = 3,// тут запятая
                Name = "Женя"// тут запятая не обязательна но можно
            };// точка с запятой
```

> Таким способом мы можем заполнить все `public` поля класса

![[Pasted image 20251202114546.png]]
> Отличие двух способов заключается в том, что  в первом:
> 	1) Мы сначала создаем пустой экземпляр класса
> А во втором:
> 	2) Мы инициализируем объект и заполняем поля одномоментно

# Зачем нужен такой синтаксис?(пример)

![[Pasted image 20251202115336.png]]

> У нас есть `class Person`, одно из полей которого это тип данных `Adress`

![[Pasted image 20251202115429.png]]

> Тип данных `Adress` включает в себя поля Страна, Регион, Город

## Инициализация обычным способом:

```csharp
            Person person1 = new Person();

            person1.FirstName = "Паша";

            person1.LastName = "Шарапов";

            Adress adress = new Adress();

            adress.Country = "Russia";

            adress.Region = "Kaluga Oblast";

            adress.City = "Kaluga";

            person1.Adress = adress; 
```

![[Pasted image 20251202120646.png]]

> То есть  у Класса `Person` есть поле имя, фамилия и поле `Adress`, внутри которого еще 3 поля: страна, регион город

## 2й способ инициализации:

```csharp
Person person1 = new Person
{
    FirstName = "Паша",
    LastName = "Шарапов",

    Adress = new Adress
    {
        Country = "Russia",
        Region = "Kaluga Oblast",
        City = "Kaluga"
    }
};
```

> это удобнее и понятнее, а в первом случае мы можем забыть привязать переменную адреса к полю класса персон `person1.Adress = adress; `

# Инициализация и конструктор

> Вернемся к классу `Cat`

```csharp
    class Cat
    {
        public int Age { get; set; }
        public string Name { get; set; }

        public Cat()
        {
			
        }
        public Cat(string name)
        {
            Name = name;
        }
    }
```

> При инициализации по умолчанию используется конструктор без параметров.

```csharp
Cat cat1 = new Cat
{
    Name = "Вася",
    Age = 1,
};
```

> Использование конструктора с параметрами при инициализации:

```csharp
Cat cat2 = new Cat("Барсик")
{
    Age = 5,
};
```

>Пример инициализации объекта класса `Person` с конструктором
((класс + конструктор)
```csharp
    class Person
    {
        public string FirstName { get; set; }
        public string LastName { get; set; }
        public Adress Adress { get; set; }

        public Person(string FirstName, string LastName) 
        {
            this.FirstName = FirstName;
            this.LastName = LastName;
        }
    }
```

> Удобная инициализация

```csharp
            Person person1 = new Person("Паша","Шарапов")
            {
                Adress = new Adress
                {
                    Country = "Russia",

                    Region = "Kaluga Oblast",

                    City = "Kaluga"
                }
            };
```

Важный момент, даже если у нас есть конструктор, но внутри инициализации мы всё равно можем изменить поля:

```csharp
Cat cat2 = new Cat("Барсик")
{
    Name = "Дима",
    Age = 5,
};
```
