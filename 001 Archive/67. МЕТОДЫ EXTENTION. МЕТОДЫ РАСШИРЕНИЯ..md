# Что такое extention методы?

> Суть: позволяют добавить новый функционал к существу

# Когда это важно и зачем нужно?

> Класс который мы модифицируем может быть написан другим разработчиком и находиться в библиотеке класса и отредактировать мы его не можем.

>Это относится и к структурам.

> Функционал extention методов есть и у наследования. Но если это структура, то у нее не бывает наследования.

> У класса от которого мы хотим наследоваться может быть модификатор `sealed`, что не позволяет наследоваться.

>Даже если мы можем унаследоваться от класса в библиотеке, это не всегда рационально, и приводит к путанице.

# Синтаксис `extention` методов.

```csharp
            DateTime CurrantDateTime = DateTime.Now;
            CurrantDateTime.Print();
```

> Напишем `extention` метод для структуры  `DateTime` который будет выводить время на консоль.

>1) Методы расширения должны находится в `static` классе и должны быть статическими

```csharp
static class MyExtantoins
{
    public static void Print(this DateTime dateTime)
    {
        Console.WriteLine($"{dateTime}");
    }
}
```

> Среда разработки подсвечивает метод как расширение

> Ключевое тут, что мы используем слово `this` и указываем передаваемый тип данных.

# Параметры принимаемые методами `extention`

>Мы можем передавать параметры методам расширения, если это предусматривает 
метод.

> Первым идет параметр с ключевым словом `this`, основной для метода расширения

```csharp
CurrantDateTime.Print();
```

# Изоляция методов расширения

> Нужна чтобы не засорять общее пространство имен своими методами расширения

```csharp 
namespace MyExtantoins
{    
	static class MyExtantoins
    {
    
        public static void Print(this DateTime dateTime)
        {
            Console.WriteLine($"{dateTime}");
        }

    }
}
```

> А в `Main` добавляем 

```csharp
using MyExtantoins;
```
