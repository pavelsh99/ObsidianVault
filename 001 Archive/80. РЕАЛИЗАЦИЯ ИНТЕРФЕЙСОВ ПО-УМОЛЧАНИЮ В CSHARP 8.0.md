# Что такое реализация интерфейсов по умолчанию и почему это спорно?

>В C# 8.0 добавлена фича, с помощью которой можно писать реализацию интерфейса прямо в нем.

> Это спорно, потому что смешивается абстракция и реализация.

![[Pasted image 20260216124719.png]]

В C# по умолчанию нет множественного наследования. Но с помощью дефолтной реализации методов в интерфейсе оно по сути добавляется но в ограниченном виде

Но такое наследование ограничено, т.к у интерфейсов не может быть полей и конструкторов как у классов
