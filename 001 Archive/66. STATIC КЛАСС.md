#  Как работает статический класс

> Мы не можем создавать экземпляры такого класса.

![[Pasted image 20251113190144.png]]

> `static` класс может содержать только статические методы, поля, свойства

![[Pasted image 20251113190606.png]]

 > В `static` классе может быть только статический конструктор.
 
 > Мы не сможем использовать `this` в статическом классе.(только в `extention` методах)

# Когда использовать `static` класс

> Их используют когда нужна утилитарная вещь для обслуживания более сложной логики. 

> Хороший пример -  класс `Math`.

![[Pasted image 20251113191129.png]]

> Он содержит в себе методы для математических подсчетов

> Не нужно злоу

# Ограничение `static` методов

> `static` методы не поддерживают полиморфизм, интерфейсы, наследование.

> Поэтому у них есть проблема с внедрением зависимостей.

> Есть проблемы при тестировании таких классов

> Поэтому в статик классы группируют тематически- похожие методы с похожим функционалом.

# Пример

> Например есть класс студент

> У него есть поля Имя группа адрес и т.д.

> Их мы не можем сделать `static` ведь все данные уникальны для каждого объекта класса

> Нам нужно вывезти имя на консоль. 

> Но зачем нам создавать объект класса консоль?

> То есть контраст `student` и консоль. Студент такой один, на консоль можем вывести всё что угодно
