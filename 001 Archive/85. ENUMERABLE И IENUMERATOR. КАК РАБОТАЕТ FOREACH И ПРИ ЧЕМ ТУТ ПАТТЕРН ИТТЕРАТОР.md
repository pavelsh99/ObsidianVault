# Что такое паттерн итератор

Паттерн итератор даёт последовательный доступ к элементам коллекции, не раскрывая её внутренней структуры. Другими словами: ты получаешь возможность «перебирать» элементы, не зная, как они хранятся внутри.

```csharp
            IEnumerable<int> sequence = [1,2,3];
            foreach (var number in sequence)
            {
                Console.WriteLine(number);
            }
```

Этот паттерн итератор нужен для того чтобы получить последовательный доступ к коллекции без раскрытия ее внутреннего устройства

Интерфейсы и `Enumerable` связаны с #LINQ 
 Это язык запросов в C#, он позволяет фильтровать и обращаться к коллекциям

`IEnumerable<int> sequence = [1,2,3];` Тут мы объявляем массив

Мы указываем тип `IEnumerable` и потому что он generic, мы указываем тип коллекции `<int>`

## Интерфейсы: 

- `IEnumerable<T>`
    
    - Метод `GetEnumerator()` возвращает `IEnumerator<T>`.
	
    - Говорит: «меня можно перебрать».
	
	- `IEnumerator<T>`


- Свойство `T Current` — текущий элемент.
	
	- Метод `bool MoveNext()` — перейти к следующему элементу (возвращает true, пока есть элементы).
	
    - Метод `void Reset()` — вернуть перечислитель в начальное состояние (используется редко).
	
    - Наследует `IDisposable` (возможен `Dispose()`).

# Пример из урока:

```csharp
    internal class Program

```


```csharp
    public class NumberSequence : IEnumerable<int>
    {
        private readonly int start;
        private readonly int count;
        public NumberSequence(int start, int count) 
        {
            this.start = start;
            this.count = count;
        }
        public IEnumerator<int> GetEnumerator()
        {
            return new NumberEnumerator(this.start, this.count);
        }

        IEnumerator IEnumerable.GetEnumerator()
        {
            return GetEnumerator();
        }
    }
```


```csharp
public class NumberEnumerator : IEnumerator<int>
{
    private readonly int start;
    private readonly int count;
    private int currentIndex;

    public NumberEnumerator(int start, int count)
    {
        this.start = start;
        this.count = count;
        currentIndex = -1;
    }
    public int Current
    {
        get 
        {
            if (currentIndex < 0 || currentIndex >= count)
                throw new InvalidOperationException(
                    "Enumerator is in an invalid state");
            return start + currentIndex;
        } 
    }

    object IEnumerator.Current => Current;

    public void Dispose()
    {
    
    }

    public bool MoveNext()
    {
        if(currentIndex  + 1  < count)
        {
            currentIndex++;
            return true;
        }
        return false;
    }

    public void Reset()
    {
        currentIndex = -1;
    }
}
```

## Короткая шпаргалка (ключевые мысли)

- `IEnumerable<T>` — описание последовательности (можно перебрать).
    
- `IEnumerator<T>` — объект, который двигается по элементам (MoveNext + Current).
    
- `foreach` использует `GetEnumerator`, `MoveNext`, `Current` и вызывает `Dispose`.
    
- Начальная позиция перечислителя обычно `-1`. Это важно.
    
- `IEnumerable<T>` не гарантирует размер, индексирование или сохранение состояния.
    
- Изменение коллекции во время перебора обычно приводит к исключению.
