# Для чего нужно ключевое слово `this`

> Благодаря ему мы можем получить доступ к экземпляру класса, его полям, методам, свойствам.

>Ключевое слово `this` используется, когда есть неоднозначность в конструкторах/свойствах методах класса, в полях класса.

> `this` обращается к конкретному элементу экземпляра класса.

# Пример:

```csharp
class Student
 public Student(string lastName, DateTime birthDay)
 {
     _birthday = birthDay;
     _lastName = lastName;
 }
 {
	private string _firstName;
	private string _lastName;
	private string _middleName;
	private DateTime _birthday;
 }
```

> Тут, для во избежание путаницы мы использовали `_` перед названием полей класса.

>	Уберем `_`:

```csharp
 public Student(string lastName, DateTime birthDay)
 {
     _birthday = birthDay;
     lastName = lastName;
 }
 {
	private string _firstName;
	private string lastName;
	private string _middleName;
	private DateTime _birthday;
 }
```

# Ключевое слово `this` перегрузка конструктора.

```csharp
            public Student(string lastName, string firstName)
            {
                this.lastName = lastName;
                this.firstName = firstName;
            }
            public Student(string lastName, string firstName, string middleName, DateTime birthDay)
            {
                this.lastName = firstName;
                this.firstName = firstName;
                _middleName = middleName;
                _birthday = birthDay;
            }
```

> В двух конструкторах есть дублирование кода.
> 
> Если нам захочется изменить логику работы класса, может случиться так, что мы изменим один конструктор, а второй не будем трогать
> 
> Мы получим ошибки при вызове одного из конструкторов

> Чтобы таких ситуаций не было мы можем вызвать цепочку вызова конструктора с `this`

> Синтаксис
```csharp
public Name(переменные):this(переменные)
{
	
}
```
Пример:
```csharp
public Student(string lastName, string firstName)
{
    this.lastName = lastName;
    this.firstName = firstName;
}
public Student(string lastName, string firstName, string middleName, DateTime birthDay):this(lastName, birthday)
{
	_middleName = middleName;
	_birthday = birthDay;
}
```

То есть если передать все данные второй конструктор перешлет нас в первый и обработает их там

# Ключевое слово `this` и модификатор `static`

>`this` невозможно использовать в классах `static`

> Это происходит, т.к. в статик классах не бывает экземпляра класса, поэтому не на что указывать при помощи `this`
