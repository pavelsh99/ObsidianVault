Ключевое слово `null` можно использовать только с ссылочными типами данных.

[[38. ССЫЛОЧНЫЕ И ЗНАЧИМЫЕ ТИПЫ. СТЕК И КУЧА]]

Например мы создаем массив, но не выделяем место под него в оперативной памяти.

```csharp
int[] a; // Null
```

![[Pasted image 20250601154055.png]]

Мы создали только ссылку, а данные не создали.
>`Null` представляет из себя пустую ссылку на объект в куче(только ссылочный тип данных), который не имеет данных.

> Но если это значимый тип данных, он будет иметь дефолтное значение в ситуации, когда нет инициализации.

> ![[Pasted image 20250601154617.png]]

Идем дальше, выделяем ОЗУ для массива с помощью оператора `new`:

```csharp
int[] a = new int[10];
```

Оператор `new` выделяет место под данные и возвращает ссылку, которая и хранится в переменной `a` 
![[Pasted image 20250601154951.png]]
 Вот что происходит после присваивания с `new`:
 ![[Pasted image 20250601155040.png]]

Но что  если дальше по коду присвоим массиву `a` значение  `null`:

```csharp
int[] a = new int[10];

a = null;
```

![[Pasted image 20250601155427.png]]

>В таком случае :
>- В стеке останутся данные 
>- В куче останутся данные , которые потом удалит сборщик мусора
>- !!! Но  ссылка на эти данные в куче пропадет

В такой ситуации из-за потери ссылки происходит exeption:
![[Pasted image 20250601160300.png]]

А в такой ситуации т.к. а это значимый тип данных `a` существует только в методе , а массив это ссылочный тип, данные хранятся в куче, то произойдет потеря связи`a` исчезнет вне метода, и данные данные массива удалятся сборщиком мусора из кучи.
