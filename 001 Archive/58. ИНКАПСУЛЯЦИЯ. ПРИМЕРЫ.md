# Что такое Инкапсуляция?

> Это принцип, по которому состояния (поля, данные) скрыты и можно изменить это состояние только при помощи методов этого же объекта.

# Пример

- У нас есть пушка, нужно, чтобы автоматически стреляла.
- Нам не важно как она перезаряжается под капотом.
```csharp
    class Gun
    {
        private bool isLoaded;/// Состояние СКРЫТО

        private void Reload()/// часть поведения СКРЫТО
        {
            Console.WriteLine("Загрузка боеприпаса...");
            isLoaded = true;
            Console.WriteLine("Заряжено, готово к стрельбе");
        }
        public void Shoot()
        {
            if(!isLoaded)
            {
                Console.WriteLine("Пусто!");
                Reload();
            }
            Console.WriteLine("ВЫСТРЕЛ!");
            isLoaded = false;
        }
    }
    internal class Program
    {
        static void Main(string[] args)
        {
            Gun gun = new Gun();
            gun.Shoot();
        }
    }
```

![[Pasted image 20251020145927.png]]


> У класса `Gun` есть **состояния** и **поведение**

> **Состояния** это значение полей класса `Gun`: `isLoaded = true` или `false`

> **Поведение** это то, что класс может сделать, то есть **методы** `Shoot()`  и `Reload()`

>Пушка у нас автоматическая, то есть функционал перезарядки скрыт за модификатором `private`, а на уровне объекта класса доступен только метод `.Shoot()`


Если бы состояние `isLoaded` было бы `public`, то можно было бы сломать логику метода:

```csharp
static void Main(string[] args)
{
	gun.isLoaded = true;
	Gun.Shoot();
}
```
------------------------------------------------

> **Инкапсуляция позволяет не допустить, чтобы пользователь класса сделал с классом что-то непредусмотренное**

> Инкапсуляция это не совсем сокрытие, это возможность гарантированно использовать класс как задумано.
