# ?.  ОПЕРАТОР УСЛОВНОГО NULL

###### Синтаксис:

```csharp
объект?.Метод() или объект?.Свойство
```

Есть ситуация где мы получаем откуда то данные:

```csharp
        static int[] GetArray()
        {
            int[] myArray = { 1, 2, 3, 4, 5, 6, 7, };
            return myArray;
        }

        static void Main(string[] args)
        {
            int[] myArray = GetArray();
            Console.WriteLine(myArray.Sum());

        }
```

Если мы получим `null`, то программа упадет, потому что мы пытаемся вызвать метод `sum` из несуществующего элемента.

`?.` проверяет на `null` и если значение равно `null` то дальнейший код не выполняется и не возникает ситуации когда существует ссылка на несуществующий объект в куче.

```csharp
        static int[] GetArray()
        {
            int[] myArray = null; // тут null
            return myArray;
        }

        static void Main(string[] args)
        {
            int[] myArray = GetArray();

            Console.WriteLine("сумма = " + myArray?.Sum());// Sum не выполнится!

        }
```

> ?. можно комбинировать с ??

```csharp
        static int[] GetArray()
        {
            int[] myArray = null; // тут null
            return myArray;
        }

        static void Main(string[] args)
        {
            int[] myArray = GetArray();

            Console.WriteLine("сумма = " + (myArray?.Sum()?? 0));

        }
```

Оператор `??` проверят на `null` и если  он прав, то на консоль выводится 0.

Еще пример:

```csharp
Cat cat = null;

string name = cat?.Name ?? "Безымянная";
Console.WriteLine(name);

```
