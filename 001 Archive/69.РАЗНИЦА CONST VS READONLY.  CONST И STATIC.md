# `Const`. Как влияет на поля класса?

>Этот модификатор позволяет сделать невозможным изменение поля.

>`Const` поле всегда должно иметь какое-то значение.

```csharp
    class Myclass
    {
        public const int a = 1; // ТУТ

        public Myclass()
        {
            
        }

    }
```

> Зачем это нужно? 
> Ответ: Когда мы заранее знаем какие данные должны находиться в конкретном поле и заранее знаем, что эти данные не должны никогда изменяться.

> Пример: число PI при расчетах или хранение сообщений об ошибках в константных полях.

```csharp
        public const double PI = 3.14;

        public const string SOME_ERROR = "SomeError";
```

# Особенности `const` полей 

> Все `const` поля обозначаются капсом и с нижними подчеркиванием

```csharp
 public const string SOME_ERROR = "error";
```

>Другая особенность, что `const` поля неявно статические, но явно добавить  модификатор`static` нельзя.

# `Const` vs `Readonly`

>Как было сказано, значение в `const` должно быть присвоено еще до компиляции кода.

> Это ограничение. То есть мы должны сразу эти данные знать еще до компиляции.

>Но что если эти данные так же нельзя изменять, но узнать их мы можем только в момент выполнения программы?

# Для этих целей есть `readonly`

>  С `readonly` мы можем сразу не присваивать значение полю.

> Отличие: он не будет неявно `static`

> Но статиком его можно сделать явно.

> Значение для `readonly` можно присвоить с помощью конструктора

```csharp
    class Myclass
    {
        public readonly int _a;

        public Myclass(int a)
        {
            _a = a;
        }

    }
```

> Что будет если мы попытаемся изменить `readonly int _a` в методе `Foo()`
