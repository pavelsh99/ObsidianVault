# Пример работы `yield return`

Тут будет использован код из предыдущего урока [[85. ENUMERABLE И IENUMERATOR. КАК РАБОТАЕТ FOREACH И ПРИ ЧЕМ ТУТ ПАТТЕРН ИТТЕРАТОР]]

У нас был класс `NumberSequence`, который
+ Наследует `IEnumerable`
+ С его помощью можно было сгенерировать коллекцию чисел.
+ Точку отсчета и количество элементов можно было выбрать

```csharp
    public class NumberSequence : IEnumerable<int>
    {
        private readonly int start;
        private readonly int count;
        public NumberSequence(int start, int count) 
        {
            this.start = start;
            this.count = count;
        }
        public IEnumerator<int> GetEnumerator()
        {
            return new NumberEnumerator(this.start, this.count);
        }
        IEnumerator IEnumerable.GetEnumerator()
        {
            return GetEnumerator();
        }
    }
```

```csharp
        static void Main(string[] args)
        {
                IEnumerable<int> sequence = new NumberSequence(start: 0, count: 1999999);
                foreach (var number in sequence)
                {
                    Console.Write("1");
                }
        }
```

мы можем сделать ту же самою логику  без `foreach`:

```csharp
        static void Main(string[] args)
        {
            IEnumerable<int> sequence = new NumberSequence(start: 0, count: 1999999);
            //foreach (var number in sequence)
            //{
            //    console.write("1");
            //}
            var enumerator = sequence.GetEnumerator();
            while (enumerator.MoveNext())
            {
                Console.WriteLine(enumerator.Current);
            }
        }
```

То есть мы метод `GetEnumerator()`, чтобы получить объект который будет проверять есть ли следующий элемент коллекции, и если да, возвращать.

Работает точно так же но без синтаксического сахара.
