+ С его помощью можно было сгенерировать коллекцию чисел.


```csharp
        static void Main(string[] args)
```

мы можем сделать ту же самою логику  без `foreach`:

```csharp
        static void Main(string[] args)
        {
            IEnumerable<int> sequence = new NumberSequence(start: 0, count: 1999999);
            //foreach (var number in sequence)
            //{
            //    console.write("1");
            //}
            var enumerator = sequence.GetEnumerator();
            while (enumerator.MoveNext())
            {
                Console.WriteLine(enumerator.Current);
            }
        }
```

То есть мы метод `GetEnumerator()`, чтобы получить объект который будет проверять есть ли следующий элемент коллекции, и если да, возвращать.

Работает точно так же но без синтаксического сахара.

# Зачем же слово `yield`?

При том что метод `GetEnumerator()` и наш собственный класс, возвращающий `Enumerator`, это всё нам не нужно

И вот этот метод теперь нам тоже не нужен:
```csharp
        public IEnumerator<int> GetEnumerator()
        {
            return new NumberEnumerator(this.start, this.count);
        }
```

Наш измененный `NumberSequence` будет выглядеть так 
```csharp
    public class NumberSequence : IEnumerable<int>
    {
        private readonly int start;
        private readonly int count;
        public NumberSequence(int start, int count) 
        {
            this.start = start;
            this.count = count;
        }
        public IEnumerator<int> GetEnumerator()
        {
            for (int i = 0; i < count ; i++ )
            {
                yield return start + i;
            }
        }

        IEnumerator IEnumerable.GetEnumerator()
        {
            return GetEnumerator();
        }
    }
```

То есть мы заменили целый класс на 
```csharp
        public IEnumerator<int> GetEnumerator()
        {
            for (int i = 0; i < count ; i++ )
            {
                yield return start + i;
            }
        }
```

И это тот же самый `Enumerator` с теми же методами `MoveNext`, свойством`Current`:

То есть `yield` генерирует итератор с методом `Movenext`, свойством `Current`,  методом`Reset`.

# Пример `yield`

```csharp
public class Student
{
    public string Name { get; set; }
}
internal class Program
{
    static void Main(string[] args)
    {
        foreach (var student in GetStudentsByYield())
        {
            Console.WriteLine(student.Name);
        }
    }
    private static IEnumerable<Student> GetStudentsByYield()
    {
        yield return new Student { Name = "Boris" };
        yield return new Student { Name = "James" };
        yield return new Student { Name = "Trinnity" };
    }
}
```

у нас есть:
+ Класс `Student` с одним свойством `Name`
+ Есть метод, который возвращает коллекцию `IEnumerable` и каждый студент возвращается с помощью `yield return`
+ `Main` мы перебираем студентов и в выводим их имена

При запуске кода итератор `yield return` возвращает ровно один элемент коллекции после чего выходит из нее и "запоминает"

Поэтому с помощью `yield return` можно эффективно обрабатывать большие объемы элементов

То есть обновляется стейт, свойтсву `Current` присваивается значение итерации, которое соответствует каждому студенту и итератор двигается

## Вывод `yield return` позволяет нам экономить время, и не писать итератор руками и он играет огромную роль при оптимизации памяти

(`yield return` использует минимальное количество памяти потому что при переборе коллекции он возвращает ее элементы по одному, а не сразу всю коллекцию)

`yield return` может обрабатывать любой объем элементов коллекции, потому что они будут обрабатываться по-одному
