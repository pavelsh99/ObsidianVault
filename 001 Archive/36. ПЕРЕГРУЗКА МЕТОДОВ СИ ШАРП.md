Перегрузка метода это наличие большого количества разных реализаций метода.

Например `Console.Writeline` имеет 19 разных вариантов использования, он перегружен.

То есть имя метода одно и тоже, а количество параметров типы данных, различные

![[Pasted image 20250530200654.png]]

Пример перегруженного метода(имя одинаковое а сигнатуры разные)

>**Сигнатура метода** в C# — это **уникальное "имя" метода, определяемое его именем и списком параметров (типами и их порядком)**.

```csharp
        static int Sum (int x, int z)// отличие
        {
            return x + z;
        }

        static int Sum(int x, int y, int z)//отличие
        { 
            return x + y + z;
        }

        static int Sum (double x, double z)//отличие
        {
            return x + z;
        }
        
        static void Main(string[] args)
        {
            int result = Sum(1, 2);

            Console.WriteLine(result);
        }
```

>Если ввести  перед методом`///` то можно будет добавить описание метода

```csharp
        /// <summary>
        /// Возвращает сумму 2х целых чисел
        /// </summary>
        /// <param name="x"></param>
        /// <param name="z"></param>
        /// <returns></returns>
        static int Sum (int x, int z)
        {
            return x + z;
        }
```
